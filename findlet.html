Project Overview

Findlet is a web-based platform designed to help people recover lost wallets by securely connecting finders and owners through identity-based data matching. Acting as Product and Project Manager, I defined the product vision based on real-world issues where wallet recovery via social media is often unstructured, inefficient, and unreliable.

I led feature scoping and project planning, translating user needs into clear system flows, role definitions, and MVP priorities. I coordinated with designers and developers to ensure timely delivery while maintaining data security and user trust. The final product delivers a structured, end-to-end flow that enables safe, verified, and efficient wallet recovery.

Key Features

ID-Based Wallet Search

Allows users to search for lost wallets using identity numbers (KTP/SIM), enabling accurate and fast matching between reports and claims. This reduces reliance on unstructured social media posts and minimizes false or unrelated matches.

Structured Wallet Reporting

Enables finders to submit detailed wallet reports, including identity information and photo uploads. Structured input ensures consistent data quality, making the verification and matching process more reliable and secure.

Automated Claim Matching

A system-driven process that automatically compares finder reports and owner claims based on identity data. This feature improves efficiency by filtering valid matches early and reducing manual intervention.

Verified & Secure Handover

Contact details are shared only after a claim is verified and approved through the system. This ensures user privacy, builds trust between parties, and prevents misuse of sensitive personal information.
